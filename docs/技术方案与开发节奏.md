# JSON-Ext 技术方案与开发节奏

## 1. 文档目标
- 将 PRD 转化为可执行的工程方案。
- 明确技术栈、模块边界、数据流和风险控制。
- 提供可追踪的开发节奏与任务拆解，支持后续持续迭代。

## 2. 技术栈选型
## 2.1 前端与构建
- `Chrome Extension Manifest V3`
- `TypeScript`
- `React + Vite`
- 样式方案：`CSS Modules`（或 `Tailwind`，二选一，建议前期优先 CSS Modules）

## 2.2 核心能力依赖（建议）
- JSON 解析与格式化：原生 `JSON.parse/stringify` + 自定义错误封装
- JSONPath 引擎：`jsonpath-plus`
- 编辑器（文本模式）：`Monaco Editor`（先可用轻量文本域替代，后续升级）
- 数据校验：`zod`（可选，用于内部数据结构约束）

## 2.3 工程质量
- 代码规范：`ESLint + Prettier`
- 单元测试：`Vitest`
- 组件测试：`@testing-library/react`

## 3. 扩展架构设计
## 3.1 扩展入口
- `popup`：仅作为“打开独立工作台”的快捷入口。
- `devtools/standalone page`：承载完整工具 UI（建议 `chrome-extension://.../app.html`）。
- `content script + background(service worker)`：
  - 检测当前页面是否为 JSON 响应。
  - 满足条件时跳转/注入到工具页并传递原始 JSON。

## 4. 核心模块拆解
## 4.1 入口识别模块（Scene Router）
- 目标：区分“接口自动接管”与“独立 Tab”两种入口。
- 输出：统一的初始化数据结构 `InitialPayload`。

## 4.2 JSON 数据模型模块（Data Model）
- 目标：将原始 JSON 转为统一节点模型，支持路径索引。
- 核心能力：
  - 节点路径生成（dot + bracket）。
  - 节点类型标识（object/array/string/number/boolean/null）。
  - 节点查找与反向映射（path -> value）。

## 4.3 双栏联动模块（Split Workspace）
- 左侧：源数据编辑与节点选择。
- 右侧：选中节点详情 + 操作区。
- 联动机制：事件总线或全局状态管理（建议 `zustand`）。

## 4.4 三视图渲染模块（Tree/Text/Table）
- 左右侧各自独立维护视图状态。
- 树形视图：节点展开折叠、类型渲染。
- 文本视图：可编辑文本 + 解析错误提示。
- 表格视图：深层对象平铺展示（flatten），支持数组行化。

## 4.5 节点操作模块（Actions）
- `format` / `minify` / `escape` / `unescape`
- `jsonpath filter`
- 操作作用域：默认针对右侧当前节点值，可扩展“全局作用域”。

## 4.6 递归子视图模块（Recursive Subview）
- 触发条件：`unescape` 后可解析为合法 JSON。
- 承载方式：右侧替换当前内容 + 面包屑返回（已确认）。
- 交互一致性：子视图复用同一套双栏与三视图交互。

## 5. 关键数据流
1. 入口模块产出 `InitialPayload`。
2. 左侧加载 JSON 并构建节点索引。
3. 用户选中节点，更新 `selectedPath`。
4. 右侧根据 `selectedPath` 渲染路径、值与操作面板。
5. 执行操作后更新当前工作数据（必要时打开子视图栈）。
6. JSONPath 在当前作用域执行，输出匹配结果并高亮/替换展示。

## 6. 难点与技术策略
## 6.1 大数据量渲染性能
- 树形视图采用按需展开与虚拟滚动。
- 文本模式延迟格式化，避免每次输入全量 parse。
- 表格模式采用分页或行虚拟化。

## 6.2 深层平铺（Flatten）规则
- 路径作为列名（如 `a.b[0].c`）。
- 数组对象按行展开；缺失字段填 `null` 或空值。
- 保留原始类型信息，避免全部字符串化。

## 6.3 JSONPath 容错
- 非法表达式即时报错。
- 空结果与错误分开展示，避免误判。

## 6.4 递归子视图稳定性
- 子视图采用栈结构管理：`viewStack[]`。
- 限制最大递归层级（例如 10 层）防止异常数据导致无穷钻取。

## 7. 开发节奏（里程碑）
## M1：项目骨架与双入口打通（1 周）
- 交付内容：
  - 扩展工程初始化（MV3、构建链路、基础页面）。
  - 独立 Tab 可打开工具页。
  - 接口 JSON 可被识别并进入工具页。
- 验收标准：
  - 两种场景都可进入同一工作台。
  - 左侧可编辑输入并完成基础 JSON 校验。

## M2：双栏联动与树形模式（1 周）
- 交付内容：
  - 左右布局完成。
  - 左侧节点选择驱动右侧详情显示（路径、值）。
  - 树形图模式达到可用状态。
- 验收标准：
  - 节点选中、展开折叠稳定。
  - 右侧路径和值准确。

## M3：文本/表格模式与节点操作（1-1.5 周）
- 交付内容：
  - 左右侧独立切换文本/表格模式。
  - 格式化、压缩、转义、反转义、JSONPath 筛选完成。
  - 表格支持深层嵌套平铺。
- 验收标准：
  - 三视图切换不丢上下文。
  - JSONPath 可执行且错误可见。

## M4：递归子视图与质量收敛（1 周）
- 交付内容：
  - 右侧子视图替换 + 面包屑返回。
  - 子视图内交互与主视图一致。
  - 性能与异常处理优化。
- 验收标准：
  - 递归解析链路稳定可回退。
  - 大 JSON 操作无明显卡死。

## 9. 测试与验收策略
- 单元测试：`flatten`、路径生成、节点操作、JSONPath 包装器。
- 组件测试：双栏联动、三视图切换、子视图面包屑。
- 手工验收：
  - 标准 JSON、畸形 JSON、超深嵌套 JSON、超长字符串。
  - 递归反转义 1-5 层用例。

## 10. 风险与应对
- 风险：大 JSON 导致卡顿。
  - 应对：虚拟滚动 + 延迟计算 + 按需解析。
- 风险：JSONPath 兼容性差异。
  - 应对：固定库版本并封装统一执行层。
- 风险：递归子视图导致状态混乱。
  - 应对：视图栈模型 + 明确作用域 + 最大层级保护。
