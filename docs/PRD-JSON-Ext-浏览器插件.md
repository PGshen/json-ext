# PRD：JSON-Ext Chrome 插件

## 1. 产品定位
JSON-Ext 是一个面向开发者的 Chrome JSON 工具插件，支持接口 JSON 自动接管与独立工作台两种使用方式，提供双栏联动、多视图渲染和节点级数据处理能力。

## 2. 产品目标
- 提升 JSON 浏览、定位、处理效率。
- 降低复杂嵌套 JSON（含转义嵌套 JSON）排查成本。
- 提供统一、稳定、可扩展的交互框架，支持后续高级能力扩展。

## 3. 使用场景
### 3.1 场景 A：接口自动接管
- 用户直接访问返回 JSON 内容的 URL（API、`.json` 文件等）。
- 插件自动识别页面内容为 JSON 并在当前标签页内触发工具视图。
- 自动接管不修改地址栏，保持原始 URL。
- 自动接管不新开标签页，采用页面内覆盖展示。
- 左侧自动填充原始 JSON 内容。
- 左侧内容允许用户修改（用于调试和临时实验）。

### 3.2 场景 B：独立 Tab 工作台
- 用户通过插件入口打开独立工具页。
- 左侧初始为空，用户手动输入/粘贴 JSON。
- 左侧内容允许编辑。

## 4. 核心交互与布局
## 4.1 整体布局
- 双栏布局：左侧“源数据区”，右侧“节点详情区”。
- 左右侧均可独立切换展示模式。

### 4.2 左侧（源数据区）
- 展示完整 JSON。
- 支持节点选中、展开/折叠。
- 用户选中任意节点后，右侧同步显示该节点信息与操作面板。

### 4.3 右侧（节点详情区）
- 展示当前选中节点的：
  - 路径（Path）
  - 值（Value）
  - 操作按钮（Format/Escape 等）
- 未选中节点时可显示空态指引或默认全局信息。

## 5. 展示模式（左右侧独立可选）
每一侧都支持以下三种模式，默认树形图模式：

1. 树形图模式（默认）
- 层级可视化、节点折叠展开、类型标识。
- 样式参考：`docs/树形图模式.jpg`。

2. 原始文本模式
- JSON 原文展示与编辑。
- 支持基础语法高亮、错误提示。
- 样式参考：`docs/纯文本模式.jpg`。

3. 表格模式
- 支持对象与数组的表格化展示。
- **必须支持深层嵌套对象平铺展示**（flatten）。
- 样式参考：`docs/表格模式.jpg`。

## 6. 节点操作能力
右侧针对当前节点支持以下操作：
- 格式化（Format）：美化缩进输出。
- 压缩（Minify）：输出单行紧凑 JSON。
- 转义（Escape）：转换为转义字符串。
- 反转义（Unescape）：还原为 JSON 文本。
- 路径筛选（Filter）：基于路径筛选结果。

## 7. 路径筛选规范
- 路径筛选采用 **JSONPath** 语法。
- 支持常见表达式：
  - 根路径：`$`
  - 属性访问：`$.a.b`
  - 数组下标：`$.list[0]`
  - 通配：`$..name`、`$[*]`
  - 条件过滤（如引擎支持）：`$.items[?(@.id==1)]`
- 非法表达式需给出明确错误提示，不可静默失败。

## 8. 递归反转义与子视图
### 8.1 问题背景
部分 JSON 字段值本身是“被转义后的 JSON 字符串”，且可能多层递归嵌套。

### 8.2 功能要求
- 当用户对节点值执行反转义后，若识别为合法 JSON，可继续“打开子视图”。
- 子视图内保持与主视图一致的完整交互能力：
  - 双栏布局
  - 三种展示模式
  - 节点选择与右侧操作
  - JSONPath 筛选

### 8.3 子视图承载方式（已确认）
- 采用 **选项 A**：在右侧区域内“替换当前内容”打开子视图。
- 顶部提供面包屑导航（如 `Root > data > payload`），支持返回上级。

## 9. 数据策略
- 不做本地持久化。
- 插件关闭或刷新后，不保留输入数据与界面状态。

## 10. 非功能需求
- 大 JSON（例如 5MB 级）下基础交互可用，不出现明显卡死。
- 操作结果具备清晰反馈（成功、失败、错误详情）。
- 对非法 JSON、空输入、超深嵌套、异常字符提供可理解报错。

## 11. 验收标准（MVP）
- 能在两种入口场景正常打开并工作。
- 接口自动接管在当前 tab 内展示，且地址栏保持原 URL 不变。
- 左右侧可独立切换树形/文本/表格三种模式。
- 左侧节点选中可驱动右侧详情展示。
- 右侧五类操作可用：格式化、压缩、转义、反转义、JSONPath 筛选。
- 表格模式支持深层嵌套平铺展示。
- 递归反转义可打开子视图，且子视图交互与主视图一致。
- 子视图采用右侧替换 + 面包屑返回。
